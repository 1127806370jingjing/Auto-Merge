# J-Flash 自动化工具 — 使用说明

将 Bootloader 与 Application 合并为 HEX，并配合 J-Flash 烧录。支持**单次合并**、**批量合并**、**HEX 烧录**。

---

## 一、设置（首次使用必读）

点击左侧边栏 **「设置」** 打开全局设置，配置会保存并自动生效。

### 1. J-Flash.exe 路径

| 项目 | 说明 |
|------|------|
| **作用** | 指定 SEGGER J-Flash 可执行文件，用于烧录。 |
| **常见位置** | `C:\Program Files\SEGGER\JLink_Vxxx\JFlash.exe`（`Vxxx` 为版本号，如 V796、V788）。 |
| **操作步骤** | ① 打开资源管理器 → 进入 `C:\Program Files\SEGGER`<br>② 在 `JLink_Vxxx` 文件夹中找到 **JFlash.exe**<br>③ 在设置中点击「J-Flash.exe」输入框，选择该 exe。 |
| **其他盘符** | 若未装在 C 盘，在对应盘符的 `Program Files\SEGGER` 下按同样方式查找。 |

### 2. J-Flash 项目文件（.jflash）

| 项目 | 说明 |
|------|------|
| **作用** | 指定芯片型号、连接方式等烧录参数，合并与烧录均使用该配置。 |
| **已有工程** | 在设置中点击「Project File」，直接选择您的 `.jflash` 文件即可。 |
| **需要新配置** | ① 先**单独打开 SEGGER J-Flash**<br>② 在 J-Flash 中配置目标芯片、接口、速率等<br>③ 菜单 **File → Save project as...** 保存为 `.jflash`<br>④ 回到本工具，在「Project File」中选择该文件即可导入。 |
| **多项目** | 可保存多个 `.jflash`，本工具使用当前在设置中选中的那一个。 |

### 3. Boot Addr / App Addr

- **作用**：合并时指定 Bootloader、Application 在 Flash 中的起始地址（十六进制）。
- **默认**：Boot `0x08000000`，App `0x08020000`（请按实际工程修改）。
- 在设置中直接编辑并保存即可。

---

## 二、BOOT 与 APP 路径（可完全自定义）

- **单次合并**：点击「Bootloader」「Application」输入框，在**任意路径**下选择 BOOT/APP 文件（支持 .bin、.hex），两文件可来自不同文件夹。
- **批量合并**：选择「BOOT 目录」与「APP 目录」（任意路径），工具按文件名自动配对；输出目录可自定义，默认在 APP 目录下的 `Merged_Output`。

BOOT 与 APP 路径**无固定要求**，单次选文件、批量选目录即可。

---

## 三、功能入口

| 入口 | 说明 |
|------|------|
| **合并** | 单次选一对 BOOT + APP，合并生成 HEX，可立即烧录验证。 |
| **批量** | 选 BOOT 目录与 APP 目录，自动匹配并批量合并生成多个 HEX。 |
| **HEX烧录** | 选 HEX 所在文件夹，在列表中选要烧录的 HEX，执行烧录。 |
| **设置** | 配置 J-Flash.exe、Project File、Boot/App 地址。 |

---

## 四、使用前检查

1. ✅ 设置中已选择 **J-Flash.exe**（建议在 `C:\Program Files\SEGGER` 下按版本选择）。
2. ✅ 设置中已选择或新建 **.jflash**；新配置请先在 J-Flash 中保存再在本工具中导入。
3. ✅ **Boot Addr**、**App Addr** 与工程一致。
4. ✅ BOOT/APP 路径可任意自定义。

完成以上后即可使用「合并」「批量」或「HEX烧录」。
